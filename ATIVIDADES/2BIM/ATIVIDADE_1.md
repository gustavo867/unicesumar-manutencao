# Atividade 1 – Rastreamento e Classificação de Bugs (2º Bimestre)

Esta atividade foca em aprofundar a análise de bugs identificados na manutenção corretiva, utilizando ferramentas de gestão de projetos (GitHub Projects) para rastrear, classificar e documentar problemas de qualidade.

## Objetivos

1. Criar um **Project no GitHub** para rastreamento de bugs (Bug Tracker).
2. Identificar e registrar **5 bugs reais** do repositório como issues.
3. Classificar cada bug em uma das categorias de defeitos.
4. Descrever o **impacto** de cada bug no sistema.
5. Identificar a **causa raiz provável** de cada bug.

## Tarefas

### 1. Criar Project no GitHub

- Criar um novo project no repositório do fork da disciplina
- Nome sugerido: **Bug Tracker**
- Configurar como projeto de rastreamento (Kanban, Table ou Timeline)

### 2. Criar Issues para 5 Bugs

As issues devem documentar bugs trabalhados na **Atividade 2 do 1º Bimestre** ou novos bugs identificados durante exploração do código.

Cada issue deve incluir:
- **Título**
- **Descrição**

### 3. Classificar os 5 Bugs

Atribua exatamente **uma categoria principal** a cada bug.

### 4. Descreva Impacto

### 5. Identificar Causa Raiz

Atribua exatamente **uma causa raiz provável** a cada bug.
